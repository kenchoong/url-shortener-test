# API Docs

This folder is reserved for generated and hand-maintained API documentation for Courtside Backend.

## Intended contents

- full OpenAPI output for the backend
- any per-module API specs
- any checked-in Postman collection or client artifacts

## Current state

- OpenAPI artifacts are generated by `npm run generate:api`
- Import `docs/api/openapi.json` or `docs/api/openapi.yaml` into Postman if you want a local API definition file instead of browsing the live Swagger UI
- The current artifacts describe the URL creation and redirect routes, including query-or-insert behavior for `POST /shorten_url`, the `short_url` field in its response, and the `400` validation response for invalid URLs
- The runtime Swagger UI is available at `/api-docs` when the app is running

## Maintenance rule

When backend routes are added or changed, update the repo's API generation flow and keep the artifacts in this folder in sync with the implementation.
